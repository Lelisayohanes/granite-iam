Granite-IAM Documentation

A modern, scalable Identity and Access Management platform built with Spring Boot, PostgreSQL, Redis, and Kafka — designed for enterprise-grade security and developer experience.

⚡ Get Started 🛠 View Architecture 📦 Explore Modules

Key Features

🔒

OAuth2 & JWT Security

Built-in OAuth2 resource server with JWT token validation, supporting fine-grained role and scope-based access control.

🚀

Event-Driven Architecture

Apache Kafka integration for asynchronous event processing, audit logging, and decoupled service communication.

🗃

Multi-Layer Design

Clean separation of concerns with Core, Application, Infrastructure, and Web modules following DDD principles.

⚡

Redis Caching

Reactive Redis integration for high-performance session management, token caching, and rate limiting.

📊

Observability

Prometheus metrics, Spring Boot Actuator health checks, and structured logging for production monitoring.

🛠

Developer Experience

Hot reload with DevTools, comprehensive OpenAPI documentation via SpringDoc, and Docker Compose for local development.

Quick Start

💡 For the complete setup guide including Docker configuration and environment variables, see Local Setup.

# Clone the repository
git clone https://github.com/Lelisayohanes/Granite-IAM.git
cd Granite-IAM

# Copy and configure environment variables
cp .env.example .env

# Start infrastructure services (PostgreSQL, Redis, Kafka)
docker-compose up -d

# Build and run the application
./gradlew :granite-iam-web:bootRun

Project Structure

Granite-IAM/
├── granite-iam-core/             # Pure domain model — no framework dependencies
│   └── User.java                 # Domain entities & value objects
├── granite-iam-application/      # Use cases & business logic
├── granite-iam-infrastructure/   # Persistence, messaging, external integrations
├── granite-iam-web/              # HTTP layer — controllers, security, Spring Boot entry point
│   └── GraniteIamApplication.java
├── docs/                         # This documentation site
├── docker-compose.yml            # Local infrastructure services
└── build.gradle                  # Root Gradle build (multi-module)
